Spring 2011

Basic Program Builder

This program is designed to help new computer science students learn the basics of how programs are structured in Java. The user can construct classes, methods, variables, and inline comments and export the file to a .java file that will successfully compile. Key words such as Statics and datatypes such as int, double, String, ect... are selected from a list to prevent compilation errors and to show the user a sample of the possible methods and variables that can be created in the format they need to be in. Methods are empty and the exported programs will not do anything at runtime, the point of this program is to demonstrate structure, not functionality.

Dungeons and Dragons Character Creation Tool

This was a team project where a user creates a valid Dungeons and Dragons 5th edition player character, presenting the user with relevant information on each step of the process and allows the user to print a hard copy of the finalized character sheet. Some attributes of the player characters are customizable, and the software allows users to add custom classes, races, abilities and skills to the system for use by player characters. The software stores the characters on a MySQL database owned by the University of North Carolina at Wilmington and allows for users to retrieve and edit characters at any time. The team consisted of myself and two others Thomas and Victor, I was responsible for building the interface and logic of the panels where the user inputs character information as well as the character sheet printing and preview. Thomas was responsible for building the database functionality and panels the provide the user information during character creation and Victor was reponsible for the panels that allwed users to add custom classes, races, abilities and skills for character creation. This code also includes the SQL file (create_project_tables.sql) that was used to construct all the relations in the database.

Age Progression algorithm validation script

The first major project I built with python was a script designed to quantitatively evaluate the age progression algorithms designed by the Institute for Interdisciplinary Studies in Identity Sciences at the University of North Carolina at Wilmington. The faces of subjects are parameterized and synthesized using Active Appearance Models. Images of subjects at different ages are compared against each other using distance measures between age-progressed parameterizations as well as a similarity score between the two images using facial recognition algorithms. Three dimensional human facial representation tool set

A continuing Python project I started in school is a 3D human face representation and manipulation toolset. The system is built with Python 2.7, a QT interface, and OpenGL for 3D rendering. It starts with a template face mesh where every vertex has been selected for a specific facial region in mind (for example any vertex making up the nose of the template face, will make up the nose of any subject face). 3D point clouds taken from scans of human subjects are registered through a landmarking procedure and 1040 vertices (the number of vertices that make up the template mesh) that best summarize the face shape are taken and used to warp the template mesh into the shape of the subjects face. The lower vertex count allows for easy manipulation, lower processing cost, and better understanding of face structure as all warped vertices belong to known facial regions.
